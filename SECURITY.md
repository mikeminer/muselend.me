# Security policy

MuseLend is pre-audit testnet software and must not hold real funds. Do not deploy the
current contracts to mainnet.

Report suspected vulnerabilities privately to the repository owner through GitHub's
private vulnerability reporting once enabled. Do not include private keys, seed phrases,
personal data or live exploit transactions. Expected acknowledgement and remediation
SLAs must be established before public testnet promotion.

No MuseLend contributor will ask for a seed phrase. User transactions are signed in the
user wallet. Mainnet administration requires a separately reviewed Safe and timelock.
